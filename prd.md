## 1. Project Overview

Farm Guardian is a proof-of-concept mobile application for dairy farm management, inspired by
JioGau Samriddhi and Afimilk's Afi2Go Prime. It uses a synthetic data engine to simulate real-world
IoT sensor readings, enabling a full working demo without any physical hardware.
**App Name** Farm Guardian
**Version** 2.0 (Proof of Concept)
**Frontend** React Native (Expo)
**Backend** Python — FastAPI
**Database** Firebase Firestore
**Authentication** Demo Auth — hardcoded credentials, no real auth service
**User Roles** Single role — Farm Manager (all permissions)
**Data Mode** Fully synthetic — Python backend generates all sensor data
**Reference Apps** JioGau Samriddhi · Afimilk Afi2Go Prime

## 3. Authentication

Authentication is demo-only. A hardcoded credential pair is used to gate access to the app. No
Firebase Auth, no token management, no registration flow — just a simple login gate to simulate the
real experience.
**Demo Username** farm@Farm Guardian.app
**Demo Password** herd


**Implementation** AsyncStorage saves a boolean isLoggedIn flag on successful login
**Session** Persists until user taps Logout; app re-opens to Home if flag is set
**Security** None required — this is for demo purposes only
**Login Screen**

- App logo + name centered at top
- Email field and password field with show/hide toggle
- Login button in Primary green — full width
- On wrong credentials: inline error message in Red below the button
- No 'Forgot Password', no 'Register' — demo only

## 4. User Role

There is a single user type: Farm Manager. This user has unrestricted access to every screen and
every action in the application. No role-based restrictions, no permissions matrix.
**Role** Farm Manager
**Permissions** Full read and write access to all modules
**Can do** Add / edit / delete cattle, log milk sessions, log AI events, dismiss alerts,
add health events, confirm calving

## 5. Feature Modules

#### 5.1 Dashboard (Home Screen)

The first screen after login. Provides a live snapshot of the whole herd. Inspired by Afimilk's ABC
Dashboard and JioGau Samriddhi's home alert feed.
**Summary Cards — top strip, 2×2 grid**

- Total Cattle on farm — tapping navigates to Cattle List
- Cattle in Heat Today — amber badge count, tapping navigates to Heat Detection
- Active Health Alerts — red badge count, tapping navigates to Alerts List
- Today's Total Milk Yield (litres) — tapping navigates to Milk Summary
**Active Alerts Feed**
- Scrollable list of all active alerts, sorted: Critical first, then Warning, then Info
- Each card: cattle name + tag, alert type, short description, time raised, Dismiss button
- Alert types generated by backend: Heat Detected, Fever, Low Rumination, Milk Drop,
Calving Due, Irregular Activity


**Herd Activity Donut Chart**

- Segments: Active / Resting / Eating / Ruminating — colored with the status color scheme
- Center label shows total head count
- Tapping a segment filters Cattle List to that activity state

#### 5.2 Cattle Registry

Master directory of all cattle. Every other module links back to individual cattle records here.
**Cattle List Screen**

- Card list: Tag ID, Name, Breed, Lactation Stage, current Status chip (color-coded)
- Search bar — filter by tag or name
- Filter pill row — filter by: Status (Healthy / Alert / In Heat / Pregnant / Dry)
- Floating green + button to add a new cattle record
**Cattle Detail Screen — tabs**
- Overview tab: Tag ID, Name, Breed, Date of Birth, Weight, Status chip
- Live Readings panel (refreshes every 30s from Firestore):
◦ Body Temperature (°C) — color coded: green normal, amber/red for anomaly
◦ Activity Level chip — Low / Normal / High
◦ Rumination Time last 24h (hours) — progress bar style
◦ Eating Duration last 24h (hours) — progress bar style
- Health Log tab — chronological health events for this animal
- Milk Records tab — 30-day line chart + session list
- Reproduction tab — heat events, AI log, pregnancy status timeline
**Add / Edit Cattle Form**
- Fields: Tag ID, Name, Breed (dropdown), Date of Birth (date picker), Weight (kg), Notes
- Inline validation — Tag ID must be unique
- Delete option (with confirmation dialog) on the Edit screen

#### 5.3 Health & Alerts

Tracks health indicators and surfaces anomalies as actionable alerts. Core module inspired by both
JioGau Samriddhi's early disease detection and Afimilk's behavioral irregularity monitoring.
**Alerts List Screen**

- Tabs: Active | Resolved | All
- Each card: cattle name, alert type, severity badge (Critical / Warning / Info), time raised,
    Dismiss button


- Tapping a card opens Alert Detail with a short recommended action description
**Alert Types (auto-generated by synthetic engine)
Alert Trigger Condition Severity**
Fever Detected Body temp > 39.5°C for 2+ ticks Critical
Hypothermia Risk Body temp < 37.5°C Critical
Irregular Activity Activity: Low for 12+ continuous hrs Warning
Low Rumination Rumination < 5 hrs in 24h window Warning
Milk Drop Yield decline > 20% vs 7-day avg Warning
Heat Detected Heat score > 75 (cycle model) Info
Calving Due Pregnancy days >= 270 Info
**Health Log (per cattle, from the Detail screen)**
- Event types: Vaccination, Treatment, Vet Visit, Illness Note, General Observation
- Add Event form: Event Type, Date, Notes, Treated By
- List view sorted newest-first

#### 5.4 Milk Production

Tracks daily milk yield per cow and surfaces production trends.
**Herd Milk Summary Screen**

- Today's total yield (litres) — large metric display in Primary green
- 7-day bar chart (total herd yield per day) — bars in Secondary green
- Top 3 producers and Bottom 3 producers for today
**Per-Cow Milk Detail (from Cattle Detail → Milk Records tab)**
- 30-day line chart for that animal — color coded: green = stable, amber = declining
- Trend badge: Stable / Rising / Declining (based on last 7d vs prior 7d average)
- Session log list: Date, Session (Morning / Evening), Yield (litres)
**Log Milk Session**
- Form: Date (default today), Session (Morning / Evening / Night), Yield in litres, Notes
- Backend auto-generates sessions for demo; manual entry overrides synthetic value

#### 5.5 Reproduction & Heat Detection


Monitors estrus cycles and pregnancy. This is the flagship feature of JioGau Samriddhi; Afimilk
achieves it via AfiCollar. Farm Guardian simulates both with the synthetic heat score engine.
**Heat Detection Screen**

- List of cattle currently in heat or predicted to enter heat within 48 hours
- Each row: cattle name, last heat date, days since last heat, Heat Score bar, Optimal AI
    Window badge
- Action buttons per row: Log AI Event | Dismiss
**Pregnancy Tracker Screen**
- List of confirmed pregnant cattle: name, conception date, expected calving date, days
remaining, stage chip (Early / Mid / Late)
- Confirm Calving button — logs calving event, resets reproductive cycle for that animal
**Log AI Event (Insemination)**
- Form: Date, Cattle (pre-filled if opened from Heat Detection), Bull/Semen Reference, Notes
- Automatically marks the animal as Pregnant (pending confirmation) and resets heat cycle
**Reproduction History (from Cattle Detail → Reproduction tab)**
- Timeline: Heat events → AI events → Pregnancy confirmation → Calving

## 6. Synthetic Data Engine (Backend)

A Python / FastAPI service generates all sensor data. An APScheduler job runs every 30 seconds
(configurable), writes readings to Firestore, and evaluates alert conditions. The React Native app
subscribes to Firestore real-time listeners — no polling required.
**6.1 Data Generated per Cow per Tick
Field Generation Logic Normal Range**
Body Temperature Gaussian noise around 38.8°C; random fever
spike injected ~5% of ticks

**6.2 API Endpoints (FastAPI)**

- GET /cattle — list all cattle with latest readings
- POST /cattle — create a new cattle record
- GET /cattle/{tag_id} — full cattle detail with history
- PUT /cattle/{tag_id} — update cattle record
- DELETE /cattle/{tag_id} — remove cattle
- POST /cattle/{tag_id}/milk — log a milk session
- POST /cattle/{tag_id}/ai-event — log insemination event
- POST /cattle/{tag_id}/calving — confirm calving
- POST /cattle/{tag_id}/health-event — add health log entry
- GET /alerts — list all active alerts
- PATCH /alerts/{alert_id}/dismiss — dismiss an alert
- POST /admin/tick — manually trigger one data generation tick (for demo)

## 9. Non-Functional Requirements

- **Dashboard and Cattle List must render within 2 seconds on Wi-Fi:** Screen load time
- **Live readings panel on Cattle Detail updates every 30 seconds via Firestore listener:**
    Data refresh
- **Synthetic data must have realistic day/night patterns; no impossible values (e.g.,**
    **negative milk yield):** Demo realism
- **Layouts must render correctly on screens 5" to 6.7":** Responsiveness
- **Any primary action reachable within 2 taps from the Dashboard:** Navigation
- **React Native project split into feature folders (auth / cattle / alerts / milk / repro);**
    **backend has a README with setup and demo run instructions:** Code structure
